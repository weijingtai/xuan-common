# 标题行参与重排（EditableFourZhuCardV3）

## 项目规划
- 目标：允许标题行（索引0）作为重排来源，插入到数据区（索引≥1）。
- 范围：仅修改 `EditableFourZhuCardV3.dart` 行重排逻辑与插入接收层；不涉及列逻辑与其他组件。
- 时间节点：
  - 规划与确认：2025-10-31
  - 实施与自检：2025-10-31

## 实施方案
- 代码调整：
  - `_reorderRows(fromAbsIdx, insertIndex)`：允许 `fromAbsIdx==0`；插入索引仍钳制到 `[1..rows.length]`，避免插入到索引0。
  - `_reorderRowsByTitlePayload`：移除对 `<=0` 的跳过约束，仅在未找到时返回。
  - `_rowInsertTarget`：移除对 `data.item2 == 0` 的拦截，保持行为一致。
- 约束：
  - 不支持删除标题行（索引0）。
  - 插入索引范围保持为 `[1..n]`，不会将任何行插入到索引0。

- 拖拽源为标题行（索引0）时：可以在插入条位置落入索引≥1的任意位置，顺利完成重排。
