# 空亡行拖拽Bottom Overflow问题修复

## 项目规划

### 问题描述
在EditableFourZhuCardV3中，当拖拽"空亡行"到卡片中时，会出现"bottom overflowed 16 px"的渲染错误。

### 修复目标
1. 分析并定位问题根源
2. 修复索引映射错误
3. 确保拖拽功能正常工作，无overflow错误

## 实施方案

### 技术分析
1. **问题根源**：`_insertExternalRow`方法在插入新行后，没有正确更新`_rowHeightOverrides`中后续行的索引映射
2. **影响范围**：所有外部行插入操作，特别是"空亡行"拖拽
3. **修复方案**：在插入新行时，正确更新所有后续行的索引映射

### 修复实现
- 文件：`/Users/jingtaiwei/Git/codex/xuan/common/lib/widgets/EditableFourZhuCardV3.dart`
- 方法：`_insertExternalRow`
- 修复内容：添加索引重映射逻辑

```dart
// 更新行高覆盖索引：插入新行后，所有后续行的索引都需要向后移动
final Map<int, double> updatedOverrides = {};
for (final entry in _rowHeightOverrides.entries) {
  final idx = entry.key;
  final height = entry.value;
  if (idx >= target) {
    // 后续行索引向后移动一位
    updatedOverrides[idx + 1] = height;
  } else {
    // 前面的行索引不变
    updatedOverrides[idx] = height;
  }
}

// 持久化行高覆盖：用于后续内部重排的反馈与占位高度一致性
final double overrideH = _rowHeightByPayload(payload);
updatedOverrides[target] = overrideH;

_rowHeightOverrides
  ..clear()
  ..addAll(updatedOverrides);
```
