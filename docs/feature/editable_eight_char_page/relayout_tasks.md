# 卡片样式编辑页 Relayout 任务清单

## 目标
- 去除重复入口：页面顶部（workspace 上方）的 save/reset/copy/delete 等按钮全部移除，仅保留页面底部命令栏作为唯一操作入口。
- 去除名称编辑重复：名称/描述输入统一收敛到 workspace 顶部的 Card 信息区；预览区不再提供第二套名称编辑入口。
- 提升信息层级：顶部只做导航与状态；workspace 顶部做元信息输入；左侧做样式编辑；中间做预览交互；右侧做预览设置；底部做命令。

## ToDo List

### P0 - 去重与结构调整（必须先做）
- [ ] 移除 workspace 上方按钮区（新建/复制/另存为/删除/保存等）并确认功能不再重复出现
  - 验收：页面顶部不再出现与底部命令栏重复的操作按钮；底部命令栏仍可完成所有操作。
- [ ] 统一“名称/描述”编辑入口，取消预览区内的名称编辑（铅笔/可编辑入口）
  - 验收：名称/描述只能在 workspace 顶部 Card 中编辑；预览区不出现可编辑名称入口（最多只读显示或完全不显示）。
- [ ] 明确顶部 AppBar 的职责为“导航 + 页面级状态（未保存）+（可选）模板选择”，不承载命令
  - 验收：AppBar 不包含保存/复制/删除/重置等命令按钮；未保存状态能被清晰感知（例如小点/标签/banner）。

### P1 - Workspace 顶部 Card 信息区（布局落地）
- [ ] 新增 workspace 顶部信息区：采用 Card 布局承载输入（两张 Card，宽屏并排、窄屏上下）
  - Card A：模板
    - [ ] 模板下拉（当前模板）
    - [ ] （可选）模板来源/分组信息展示
  - Card B：卡片信息
    - [ ] 名称（TextField，单行）
    - [ ] 描述（TextField，多行，可选）
  - 验收：名称/描述在 Card 内呈现，布局稳定；窗口变窄时自动换为上下排版且不溢出。
- [ ] 将“未保存更改”提示条与 Card 信息区/顶部状态联动（保留一种主要提示形式）
  - 验收：未保存状态在页面中只以一种主要方式呈现（banner 或 AppBar 状态），避免重复提示。

### P2 - 底部命令栏规范化（唯一命令入口）
- [ ] 底部命令栏做清晰分组与可用性状态
  - 左侧：撤销 / 重做
  - 中间：保存更改（主按钮）
  - 右侧：放弃更改（Reset）/ 复制 / 删除
  - 验收：dirty=false 时保存/放弃禁用；dirty=true 时保存可用；撤销/重做随历史栈启用/禁用。
- [ ] 删除操作增加二次确认（明确删除对象：模板/配置）
  - 验收：删除必须二次确认；文案明确删除影响范围；确认后状态与列表刷新正确。

### P3 - 右侧“预览设置”面板整理（语义收敛）
- [ ] 右侧开关区按“预览设置”分组并只影响预览/辅助，不含模板命令
  - 工作区：明/暗
  - 预览模式：纯色/彩色/黑白
  - 辅助显示：显示抓手；（Debug）滞回边界
  - 验收：右侧面板不出现保存/复制/删除等按钮；各开关影响范围清晰且一致。
- [ ] Debug 开关仅在 Debug/开发环境展示（若当前已有 Debug 标识）
  - 验收：Release 不出现调试开关；Debug 下可用。

### P4 - 预览区体验（不引入新入口前提下增强反馈）
- [ ] 预览区顶部轻量工具条（可选）：缩放/Fit、居中、重置视图（不包含命令）
  - 验收：工具条只影响预览呈现，不引入保存/复制等入口。
- [ ] 拖拽操作完成提示（toast/snackbar），例如“已移动：年 → 月之后”
  - 验收：成功重排/插入后出现短提示；失败/不允许放置也有明确提示。

## 边界与约束
- 不新增第二套命令入口；所有保存/重置/复制/删除只保留底部命令栏。
- 不在预览区放置名称/描述编辑控件，避免与 Card 信息区重复。
- 若“模板名称”与“卡片名称”在业务上是两个不同概念，必须通过文案与字段命名区分；否则视为同一字段并合并入口。

## 验收检查清单（最终）
- [ ] 顶部 workspace 区域不再出现重复的 save/reset/copy/delete
- [ ] 名称/描述只有一个可编辑入口（Card 信息区）
- [ ] AppBar 不承载命令按钮，仅导航与状态
- [ ] 底部命令栏作为唯一操作入口，状态联动正确
- [ ] 预览区只负责预览与交互，未引入重复入口
