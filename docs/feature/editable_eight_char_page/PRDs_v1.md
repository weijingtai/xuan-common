# Product Requirements Document (PRD): Editable Four Zhu Card System
> **Version**: 1.0 (Derived from Codebase)  
> **Status**: Implementation / Refinement Phase  
> **Related Feature**: `FourZhuEditPage`, `EditableFourZhuCard`

## 1. 概述 (Overview)
本产品模块旨在提供一个高度可定制的“四柱八字”排盘卡片编辑器。允许用户（易学爱好者、专业命理师）通过可视化界面调整八字卡片的布局、内容显示、配色与字体样式，并将这些配置保存为模板以便在不同场景下复用。

### 1.1 核心目标 (Core Goals)
*   **所见即所得 (WYSIWYG)**: 用户在编辑样式的同时，能实时预览真实数据的渲染效果。
*   **高度灵活**: 支持对行（Row）、柱（Pillar/Column）、单元格（Cell）维度的精细化控制。
*   **模板化管理**: 支持多套样式模板的保存、切换与管理，满足不同流派的排盘习惯。

## 2. 用户角色 (User Personas)
*   **初级用户**: 使用系统预设的几套经典模板，仅进行简单的字体大小或颜色微调。
*   **专业用户 (Power User)**: 根据特定流派（如盲派、子平法）需求，深度定制显示的行内容（如隐藏纳音、显示藏干）、调整行顺序，并创建专属的排盘样式。

## 3. 功能需求 (Functional Requirements)

### 3.1 编辑器工作区 (Editor Workspace)
工作区是用户进行交互的核心区域，承载预览与全局控制功能。

*   **实时预览 (Live Preview)**:
    *   渲染 `EditableFourZhuCard` 组件，展示年、月、日、时四柱数据。
    *   数据源：支持使用默认测试数据（甲子年/月/日/时）或自定义输入的八字数据进行预览。
*   **环境模拟控制**:
    *   **主题切换**: 独立于 App 全局主题，强制预览 Light (亮色) / Dark (暗色) 模式下的卡片表现。
    *   **色彩模式**: 切换 "多彩模式" (Colorful) 与 "纯净/黑白模式" (Pure)，模拟不同打印或显示需求。
    *   **辅助线/抓手**: 开启/关闭拖拽手柄 (Grip Handles) 的显示，方便查看最终纯净效果。

### 3.2 侧边栏配置系统 (Sidebar Configuration)
侧边栏提供分层级的样式与布局控制。

#### 3.2.1 卡片样式 (Card Style)
*   **全局容器**: 设置卡片的背景色、边框、圆角、阴影。
*   **内边距 (Padding)**: 调整卡片内容与边缘的距离。
*   **分隔线**: 设置行与列之间分隔线的颜色与粗细。

#### 3.2.2 行样式 (Row Style) - *核心布局功能*
*   **可见性控制**: 开关特定行的显示（例如：天干、地支、十神、藏干、纳音、空亡、表头等）。
*   **排序 (Reorder)**: 通过拖拽调整行的上下顺序（如将“十神”行移动到“天干”行上方）。
*   **行属性配置**:
    *   **标题显示**: 决定是否在单元格内显示行标题（如“正官”字样）。
    *   **对齐方式**: 左对齐、居中、右对齐。
    *   **字体样式**: 针对该行特定内容的字体大小、颜色、粗细设置。

#### 3.2.3 柱样式 (Pillar/Column Style)
*   **列宽控制**: 调整年柱、月柱、日柱、时柱及标题列的宽度比例。
*   **列属性**: 针对特定列（如“日柱”）设置特殊的背景高亮或边框，以强调日主。

#### 3.2.4 单元格样式 (Cell Style)
*   提供最细粒度的控制，针对特定位置的单元格进行样式覆盖（如“日支”单元格）。

### 3.3 模板管理系统 (Template Management)
*   **CRUD 操作**:
    *   **新建**: 创建空白模板或基于现有模板复制。
    *   **保存**: 将当前 JSON 配置持久化存储到本地数据库。
    *   **读取**: 加载已保存的模板列表。
    *   **删除**: 移除不再需要的模板（需二次确认）。
*   **分组管理**:
    *   支持创建分组文件夹，将模板归类。
    *   支持删除分组。
*   **历史记录 (History)**:
    *   **撤销 (Undo)**: 回退上一步样式修改。
    *   **重做 (Redo)**: 恢复上一步撤销的操作。
    *   **脏状态检测**: 当有未保存修改时，通过 UI 提示用户（Banner 或 按钮状态）。

## 4. 交互与 UI 规范 (Interaction & UI Specifications)

### 4.1 布局结构
*   **Top Bar**: 包含返回按钮、当前模板名称显示、保存快捷键、撤销/重做按钮。
*   **Left Sidebar (320px)**: 放置 `SidebarExplorer`，采用折叠面板 (ExpansionTile) 组织各级配置项。
*   **Center Stage**: 放置预览卡片，支持居中显示与自适应缩放。
*   **Bottom Panel (Optional)**: 用于放置快捷标签栏 (Tag Bar) 或其他辅助工具。

### 4.2 拖拽交互
*   在“行样式”面板或预览区（如果支持），用户可以通过拖拽手柄图标 (Drag Handle) 直观地交换行位置。
*   拖拽过程中需提供视觉反馈（Ghost Widget 或 占位符）。

## 5. 技术约束与数据模型 (Technical Constraints & Data Models)

### 5.1 数据存储
*   模板数据以 JSON 格式序列化存储。
*   核心模型：`LayoutTemplate` (包含 `RowConfig` 列表, `CardStyleConfig` 等)。

### 5.2 状态管理
*   必须实现 ViewModel 层与 View 层的解耦。
*   编辑器内部状态（如当前选中的颜色、临时的拖拽顺序）不应污染全局应用状态，除非用户点击“保存”。

### 5.3 性能要求
*   预览渲染需保持 60fps，避免因样式属性过多导致重绘卡顿。
*   样式更新应尽量做到局部刷新或高效全量刷新。

## 6. 待办事项与未来规划 (Todo & Future)
*   **国际化**: 目前界面存在硬编码中文，需接入 l10n。
*   **分享功能**: 生成模板的分享码或文件，允许用户间互传样式。
*   **云端同步**: 将本地模板同步至用户账号云端。
*   **架构重构**: 移除 `FourZhuEditPage` 对全局 `DemoViewModel` 的依赖，确保模块独立性。
