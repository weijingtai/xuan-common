# Code Review Checklist：定子时 & 定节气/物候（联动）& 交节方案（分钟/秒）

## 设计一致性
- [x] 子时策略与“日界锚点”是否统一贯穿计算（八字、节气、物候）？
- [x] 节气/物候策略是否通过统一入口（helper）读取运行期 Store，避免分散读取？
- [x] UI 中设置卡是否放置在主卡片外侧，交互是否清晰？
- [x] 交节方案（精度）与子时策略的耦合是否明示（时辰精度依赖日界规则）？

## 逻辑正确性
- 子时
  - [x] `ZiShiStrategy` 映射到 `FourZhuEngine` 的 `ZiBoundary/ChildHourMode` 正确无误。
  - [x] `Day23BoundaryStrategy/Day0BoundaryStrategy` 产生的锚点与预期一致。
  - [x] 小时策略（五鼠遁、固定子时、两小时一支）与现有实现一致。
- 节气
  - [x] 定气法：`lunar.getPrev/Current/NextJieQi()` 的使用是否覆盖“当天正值节气”的 next 指针问题（加 2 天取 next）。
  - [x] 平气法：固定间隔（回归年/24）计算方式是否清晰，起点是否明确（上一枚定气交节时刻）。
- 物候
  - [x] 候序计算是否严格以“节气起点 + n×5 天”划分（n=0/1/2），并与所选节气基准一致。
  - [x] 候列表是否从 `Phenology.phenologyList` 按节气筛选出 3 条，索引越界时有保护。
- 交节方案（精度）
  - [x] `_sameBucket/_entered` 是否准确表达四种精度（秒/分钟/小时/时辰）？
  - [x] 时辰精度下的“同桶进入”是否正确使用当前子时策略的日界规则？
  - [x] 节气边界与物候边界均应用了精度判定，且“秒”模式不会提前进入；分钟/小时/时辰为“同桶进入”。
