# PRD：定子时与定节气/物候（联动）

## 背景与目标
- 以用户可选策略的方式，完善四柱（八字）计算在“日界/子时”、“节气/物候”上的边界与一致性。
- 提供两类核心能力：
  - 定子时：多种“子时/换日”策略，确保日柱、时柱的一致锚定。
  - 定节气与物候：提供“定气法/平气法”两种节气划分，以及“基于定气/基于平气”的物候划分，并与日界策略一致对齐。

## 术语说明
- 日界锚点：用于决定“所属于哪一日”的参照时间，由子时策略产生。
- 定气法：以太阳黄经每 15° 为节气边界（间隔不等）。
- 平气法：将回归年等分为 24 段（恒定间隔约 15.2184 天）。
- 物候：每节气划分为三候，按 5 天为一候（初候/二候/三候）。

## 用户故事
1. 作为用户，我希望在 23:00–01:00 期间可以按“23:00 统一次日”简化规则排盘。
2. 作为用户，我希望按“0:00 为界（五鼠遁/固定子时/两小时一支）”等细化策略排盘。
3. 作为用户，我希望节气可以选择“定气法（现代天文）/平气法（恒定间隔）”。
4. 作为用户，我希望物候能与节气方案保持一致（联动），并能一键选择“平气法+固定物候”或“定气法+精准物候”。
5. 作为用户，我希望将上述选择“设为默认”，在下次打开时自动应用。

## 功能范围
### 一、定子时（子时/换日策略）
- 支持策略：
  - 不区分早晚（23:00 统一次日）
  - 区分早晚（0:00 为界，五鼠遁）
  - 区分早晚（0:00 为界，固定子时）
  - 全天两小时一支（子从0点起）
- 行为：
  - 对输入时间计算日界锚点 `anchor.effectiveDateTime`，并据此生成 `Lunar` 对象与四柱；
  - 所有“节气/物候”计算均以该锚点对齐，保证一致性。

### 二、定节气与物候（联动）
- 可选组：
  - 平气法 + 传统固定物候（交节时刻恒定间隔；物候起点 = 节气起点 + n×5 天）
  - 定气法 + 现代精准物候（交节时刻依黄经；物候起点 = 节气起点 + n×5 天）
- 计算：
  - 定气法：通过 `lunar` 提供的交节时刻（上一/当前/下一节气）获得真实边界；
  - 平气法：以“上一枚定气交节时刻”为锚点，叠加固定间隔（回归年/24）得到本节起止（工程近似，可替换为严谨年表）；
- 物候：依据所选“节气基准”的节气起点，按 5 天为一候，定位初/二/三候。

- 目的：控制“何时视为进入下一节气/下一候”的判定精度，兼顾术数与现代天文。
- 选项（默认推荐“按分钟”）：
  1) 按“时辰”交节：两小时为一段（时辰）。若“给定时间”与“交节时刻”处于同一时辰，则判定为已进入新节气/新一候。时辰划分受“定子时”策略影响（23 界或 0 界）。
  2) 按“小时”交节：若“给定时间”与“交节时刻”处于同一小时，则判定为已进入新节气/新一候。
  3) 按“分钟”交节（推荐）：若“给定时间”与“交节时刻”处于同一分钟，则判定为已进入新节气/新一候。
  4) 按“秒”交节：严格按真实交节时刻（精确到秒）判定是否进入。
- 应用范围：
  - 节气边界：在定气/平气两套边界上应用“同桶进入”判定（时辰/小时）；分钟/秒模式不提前进入。
  - 物候边界：对 +5 天、+10 天分界同样应用上述判定。

## UI/交互
- 设置卡片：
  - 定子时卡：`ZiStrategySettingsCard`，支持“设为默认/应用并重算”。
  - 节气与物候（联动）卡：`JieQiPhenologySettingsCard`，二选一（平气+固定/定气+精准），支持“设为默认/应用并重算”。
  - 交节方案卡：`JieQiEntrySettingsCard`，多选一（时辰/小时/分钟/秒），默认勾选“按分钟交节（推荐）”，支持“设为默认/应用并重算”。
- 位置：两张卡片放在主查询卡（时间选择）外部下方，便于快速切换并刷新。

## 数据与状态
- 运行期 Store：
  - 子时：`ZiStrategyStore.current`
  - 节气与物候（联动）：`JieQiPhenologyStore.jieQiType / phenologyStrategy`
- 持久化：`SharedPreferences`，均支持“设为默认”。

## 验收标准
- 切换“定子时”任意策略，八字日柱/时柱、节气/物候随锚点一致变化。
- 切换“节气与物候（联动）”两组选项，`JieQiInfo.startAt/endAt` 与物候候序相应变化。
- “设为默认”后重启应用仍保留选择。
- 常见边界正确：
  - 23:00/00:00 边界
  - 月末/年末
  - 当天正值交节时刻
  - 不同时区、夏令时去除后的路径、平/真太阳时路径一致。
  - 交节方案各精度在节气与物候边界的判定差异符合定义（时辰/小时/分钟“同桶进入”，秒严格时刻）。

## 风险与后续
- 平气法当前为工程近似（基于定气锚点+固定间隔），如需严格“恒气年表”可引入数据表或精确推导。
- Widget 测试需补全 Provider 注入（见仓库现有说明）。
 - “时辰”精度依赖当前子时策略的日界规则，需在文档中明确说明；测试覆盖不同子时策略下的同桶判定。
