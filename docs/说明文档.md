# 空亡行拖拽溢出问题修复说明

## 项目规划
修复"空亡行"在重新排序时出现的底部溢出问题。

## 实施方案

### 问题分析
1. **问题现象**：当"空亡行"被拖拽重新排序时，UI显示底部溢出错误
2. **根本原因**：行拖拽反馈没有进行垂直偏移，导致拖拽反馈超出 `AnimatedContainer` 的底部边界
3. **对比发现**：列拖拽使用 `_offsetFeedbackUp` 进行垂直偏移来避免溢出，但行拖拽只使用 `_offsetFeedbackRight` 进行水平偏移

### 技术实现方案
1. **修复位置**：`EditableFourZhuCardV3.dart` 中的两个行拖拽 `Draggable` 组件
2. **修复方法**：为行拖拽的 `feedback` 添加垂直偏移
3. **具体实现**：
   - 将原来的 `_offsetFeedbackRight(child, 0)` 
   - 改为 `_offsetFeedbackUp(_offsetFeedbackRight(child, 0), _rowHeightByName(rowName) / 2)`
   - 向上偏移行高的一半，防止底部溢出

### 修复的代码位置
- 第一处：行 1588-1605（行标题拖拽抓手）
- 第二处：行 1697-1714（行内容拖拽）
